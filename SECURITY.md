# Security Policy

## Scope

This project handles governance-sensitive logic and may be adapted for public safety workflows. Security and responsible disclosure are critical.

## Reporting a Vulnerability

Until a dedicated security mailbox is configured, open a private maintainer contact issue request and avoid public exploit details.

For public, institutional, or partnership inquiries unrelated to responsible disclosure, use only the contact policy in CONTACT.md.

Include:

- affected component,
- reproduction steps,
- impact assessment,
- suggested mitigation.

## Disclosure Expectations

- Do not publish exploit details before maintainers acknowledge and assess.
- Allow maintainers reasonable time to investigate and patch.
- Coordinate on remediation notes for public disclosure.

## High Priority Security Areas

- Unauthorized enforcement pathways.
- Privilege escalation in moderation tools.
- Audit log tampering risk.
- Policy bypass through malformed prompts or retrieval payloads.
- Sensitive transcript or metadata exposure.
